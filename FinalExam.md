#### GIT, VERSIONAMIENTO Y TRABAJO PROFESIONAL
   
4. Explica con tus palabras el flujo básico de trabajo en Git (commit, push, pull, branch, merge).
   
   El flujo básico en Git inicia creando cambios en el proyecto desde tu computadora.
   Cuando ya tienes avances, haces un commit, que es como guardar una versión exacta de lo que llevas.
   Después, con un push, envías esos cambios al repositorio remoto, como GitHub.
   Antes de trabajar, se recomienda hacer pull, para traer a tu computadora los cambios más recientes que hicieron otros.

6. ¿Por qué es importante usar control de versiones en proyectos reales?
   
   Porque permite llevar un historial claro de los cambios, saber quién modificó qué, recuperar versiones anteriores si algo falla y evitar que varias personas sobreescriban sus avances.
   También facilita el trabajo en equipo, la colaboración segura y la organización profesional del código o documentos del proyecto.

8. ¿Cómo aplicarías GitHub en un proyecto profesional o universitario en equipo?
   
    Usaría GitHub como plataforma central para organizar el proyecto, subir código o documentos y trabajar de forma colaborativa.
